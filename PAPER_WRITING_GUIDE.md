# 论文写作指导：MIGO 与 Joint CFR-DPD

本文档用于指导当前硬件设计的论文写作。整体表述建议以“架构设计思路清晰、模块功能边界明确、创新点突出”为主线，弱化代码级实现、脚本流程、寄存器命名和工程调参细节。写作时建议始终回答三个问题：该结构解决了什么问题、各模块分别承担什么功能、为什么这种组织方式比传统实现更适合硬件部署。

## MIGO

### 写作定位

建议将 MIGO 写成“面向发送链路线性整形的硬件友好型定系数滤波架构”。当前 RTL 原型本质上是一个高阶、对称、定系数 FIR 结构，但论文中不必强调自动生成代码或具体系数来源，而应突出“如何把离线优化得到的滤波器映射成低复杂度硬件结构”。

### 推荐叙述顺序

第一步先说明问题背景：高阶 FIR 虽然能提供较好的频谱整形与带外抑制能力，但直接型实现会带来较高乘法器开销与数据通路复杂度，不利于资源受限平台部署。

第二步介绍总体结构，可按以下链路展开：输入采样首先进入延时线缓存，构成完整的时域窗口；随后利用冲激响应对称性，对成对样本进行对称预加，先在数据层面完成冗余压缩；再将预加结果按照系数组合关系划分为若干组，使用移位与加减替代通用乘法；最后经过统一累加与定点舍入得到输出。这样写能让读者迅速把握该模块的硬件组织逻辑。

第三步说明模块功能分工：

- 延时线模块负责维持滑动窗口，保证流式输入下每个采样都能关联到完整滤波上下文。
- 对称预加模块负责挖掘线性相位 FIR 的结构冗余，将两端样本合并，降低后续运算规模。
- 分组预求和模块负责把多个常系数乘法重写为共享的移位加法网络，体现结构化算术优化。
- 最终累加与量化模块负责完成数值收敛、位宽控制和输出格式统一。

### 创新点提炼方式

建议将创新性集中概括为以下几类。

- 从“算法系数”到“硬件算子”的直接映射。论文可强调该设计不是简单把 FIR 系数搬到硬件，而是进一步重构为适合门级实现的预加和移位加法网络。
- 充分利用滤波器对称性与常系数特征。相比直接型 FIR，结构冗余被系统性消除，因此更适合低面积、低功耗实现。
- 将高阶滤波拆解为少量可解释的算术子块。这样不仅降低实现复杂度，也使设计更容易分析时序路径和量化误差传播。
- 保持流式处理特征。读者应感受到该模块不是离线块处理单元，而是可直接嵌入发送链路的在线前端。

### 建议采用的论文表述

可以将 MIGO 的核心思想概括为：该设计以高阶定系数 FIR 为基础，但并未沿用传统直接乘加实现，而是围绕冲激响应对称性与常系数结构展开硬件重构。通过“延时线 - 对称预加 - 分组移位加法 - 定点输出”这一分层数据通路，系统在保持滤波能力的同时显著降低了通用乘法资源依赖，从而提升了该类滤波器在 ASIC/FPGA 平台上的部署友好性。

如果需要强调贡献，可进一步写成：与仅关注滤波性能的算法设计不同，该工作同时关注滤波器结构可综合性，因而在方法层面完成了从频域指标优化到硬件算术重构的闭环。

### 写作提醒

- 不建议在正文中逐个列出系数组、组号或具体移位常数，这些内容可放在附录或简写为“若干分组常系数算子”。
- 不建议把重点放在自动生成脚本或单元测试流程上。
- 若需要对比基线，优先对比“直接型 FIR 实现”或“未做对称/分组优化的定系数滤波器”。

## Joint CFR-DPD

### 写作定位

建议将 Joint CFR-DPD 写成“面向功放线性化的流式联合补偿架构”。表述重点不是某个具体神经网络名称，而是“将峰均比抑制与预失真补偿统一到同一时序建模框架中”。当前原型采用量化、状态保持、ROM 参数驱动的流式硬件结构，这些都非常适合作为论文中的架构亮点。

### 推荐叙述顺序

首先说明传统串联式 CFR 与 DPD 的局限：前者主要约束峰值，后者主要补偿非线性与记忆效应，两者分开设计时往往存在目标割裂、接口耦合松散和重复开销等问题。因此本文将两类补偿需求统一建模，在单一硬件数据通路内完成联合推理。

随后给出顶层结构描述。建议把整体链路写成：输入样本以流接口进入系统后，先经过输入投影完成特征展开；中间采用多层联合补偿块进行时序建模与非线性映射；最后通过输出投影恢复目标输出。当前原型是紧凑的双层堆叠结构，但写作时可把重点放在“层级化联合补偿框架”，而不是具体层数。

接着介绍联合补偿块的内部功能，可按以下逻辑拆分。

- 时间移位/历史缓存子模块：保存多时刻输入或中间特征，为后续建模提供记忆信息，承担对 PA 记忆效应的显式表征。
- Time-Mix 子模块：将当前输入与历史状态按通道混合，生成后续注意分支和门控分支所需的表征，体现对短时动态的建模能力。
- Key/Value/Receptance 线性变换子模块：把统一特征映射为不同功能分量，使系统可以分别刻画幅度演化、记忆加权和门控抑制。
- WKV/状态更新子模块：以递推方式维护历史累积状态，用较低的状态代价表达较长时间范围内的动态关系，是整个联合架构区别于纯前馈补偿器的重要部分。
- 门控与输出映射子模块：通过门控控制补偿强度，再将补偿结果回写到主残差通路，形成逐层精化的线性化过程。
- FFN 型非线性分支：进一步对主通路残差进行非线性修正，增强对复杂 AM-AM/AM-PM 畸变的表达能力。

### 创新点提炼方式

Joint CFR-DPD 的创新性建议从以下四个角度组织。

- 联合性。论文应突出该架构并非“CFR 后接 DPD”的简单串联，而是在统一模型中共同优化峰值抑制与非线性补偿目标。
- 状态性。相比静态前馈网络，该设计显式保留历史缓存与递推状态，更适合处理功放记忆效应和跨时刻相关失真。
- 硬件友好性。系统采用 ROM 存储参数、整数定点算术和共享线性计算引擎，使联合模型能够从算法描述自然落地到流式 RTL。
- 模块可解释性。虽然整体是联合学习/联合建模框架，但内部仍可被拆解为时间混合、门控、状态更新、输出投影等功能明确的子模块，便于论文叙事和读者理解。

### 建议采用的论文表述

可以将 Joint CFR-DPD 的总体思想概括为：本文不再将峰均比抑制和预失真补偿视为两个彼此独立的串行步骤，而是构建了一个统一的流式联合补偿架构。该架构通过历史缓存与递推状态捕获时域记忆特征，通过多分支线性映射和门控机制完成峰值约束与非线性修正的协同建模，并最终以全整数、参数固化的形式实现硬件部署。

若需要进一步强调架构创新，可写成：该设计的关键不在于简单提高模型复杂度，而在于把补偿任务拆解为“时间记忆建模 + 门控调制 + 非线性映射 + 残差更新”几个硬件可解释环节，使联合 CFR-DPD 不再停留在软件模型层，而具备清晰的 RTL 架构表达。

### 写作提醒

- 不建议正文中详细展开状态机编号、ROM 地址生成方式或具体位宽参数。
- 对量化部分建议写成“全整数定点推理框架”即可，除非论文专门有一节讨论量化策略。
- 对共享线性引擎的描述建议强调“计算资源复用”，而不是逐周期调度细节。
- 如果需要说明原型规模，可在实验设置里简要给出层数与通道规模，但方法章节应保持抽象和结构化。

## 联合成文建议

如果论文同时包含 MIGO 与 Joint CFR-DPD，推荐把二者组织成“线性整形前端 + 非线性联合补偿后端”的完整发送链路。这样写有两个好处：一是逻辑上形成从频谱控制到功放线性化的自然递进；二是可以突出本文并非只优化单个模块，而是围绕发送链路可部署性进行体系化设计。

在章节安排上，可以先写系统目标，再分别介绍 MIGO 与 Joint CFR-DPD 的内部架构，最后在实验部分统一展示对频谱、峰均比、失真抑制和部署代价的综合收益。这样更容易把“创新性”从单点模块提升到整体架构层面。
